Vue

1、前导

1.1、vue.config.js 文件

vue.config.js文件是vue-cli脚手架的配置文件，这个文件是在项目的根目录。

1.2、环境变量

  • process 是node.js的进程，这里包含了很多的运行环境的信息。

  • process.env 这是叫做进程里面的环境变量，vue-cli脚手架默认注入了两个环境变量信息

  • 自定义一些环境变量

    项目根目录创建 .env 文件 ，.env.production 或者 .env.development 三个文件。

    • .env 文件表示任何环境下都可以读取到的环境变量

    • .env.development 表示开发环境下可以读取到的环境变量信息

    • .env.production 表示生产环境可以读取的环境变量信息

    • 通过 process.env.名称来读取环境变量信息，在不同的环境下读取不同的内容

    • 注意 .gitignore文件里面是否忽略提交 上述这些文件

2、项目打包

  • 项目执行打包指令 npm run build ，执行结束之后 出现dist目录，我们讲给dist目录交给后台去部署到静态目录即可

3、打包注意事项

3.1、修改相对路径【重要】

  • 默认情况下打包出来的dist目录里面的index.html文件是不能直接打开的，因为里面的静态文件引用都是绝对路径，无法获取到资源。需要部署到服务器环境下面才可以访问到。

  • 如果我们希望得到在打包之后直接能点击index.html就可以预览，需要将生产环境下路径引用关系改为相对路径。

    然后再次执行npm run build 即可出现dist目录。

    • 方式1：vue ui 工具

    • 方式2：webpack 插件分析,在vue.config.js中配置输出
